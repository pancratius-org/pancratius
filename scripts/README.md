# scripts

## import_docx.py

Author-facing importer for one DOCX into the work-bundle layout. It uses
existing Markdown frontmatter under `src/content/**/{ru,en}.md` as the catalog;
it does not read `legacy/data/*` and it does not regenerate the corpus.

New work:

```sh
uv run scripts/import_docx.py /path/to/new.docx --kind book --lang ru
```

Existing work translation:

```sh
uv run scripts/import_docx.py /path/to/book-en.docx --into 30-poslanie-musulmanam --lang en
```

Useful overrides:

```sh
uv run scripts/import_docx.py /path/to/new.docx \
  --kind book \
  --lang ru \
  --number 73 \
  --slug 73-new-work \
  --title "New Work" \
  --description "Short editorial description." \
  --cover /path/to/cover.jpg
```

By default it writes to `src/content/`. For scratch/testing, pass
`--out-content /private/tmp/pancratius-import/src/content`.

The importer creates the bundle folder, writes `<lang>.md`, writes the
cleaned/optimized DOCX artifact to `<lang>.docx`, rewrites imported body images
to `images/<hash>.<ext>`, and writes `bibliography.yaml` when the DOCX contains
a parseable catalog table. Missing descriptions are seeded as an obvious
`TODO:` value that satisfies the content schema.

After import, run the local release-artifact scripts for the changed work before
shipping: render PDF/EPUB and refresh the bulk Markdown archive manifest. CI
only validates and publishes committed artifacts.

## The DOCX→Markdown converter (typed-IR pipeline)

`import_docx.py` drives the conversion through the
`scripts/lib/docx_conversion.py` facade (`convert_single_docx`), which runs a
typed intermediate-representation pipeline: `docx_adapter` (pandoc JSON AST +
the OOXML `w:jc` side-channel → typed IR) → `ir_normalize` (structural cleanup
on the IR) → `ir_lower` (the single Markdown-lowering pass). These are library
modules under the shared `scripts/lib/` core — there is no script entry point and
no batch CLI. The converter never reads `legacy/data/*` and never writes to
`src/content`: it returns a Markdown body, sidecar bibliography data, and a list
of *planned* body assets; the importer turns those into a WritePlan and the
writer (`scripts/lib/writer.py`) is the sole component that mutates the content
tree.

### What it does

For one source `.docx`:

1. Runs pandoc to JSON, extracting media into a caller-provided directory (a
   disposable staging dir under `.cache/`, never `src/content`), and reads the
   OOXML `w:jc` paragraph alignment pandoc structurally drops (`lib/ooxml.py`).
2. Adapts the AST into the typed IR (`lib/ir.py`), then normalizes it
   (`lib/ir_normalize.py`): strips the autogenerated Word TOC block, lifts
   DOCX endmatter bibliography/catalog tables into `bibliography.yaml` sidecar
   data (per `docs/content-model.md`; their cover thumbnails never enter the
   body or the asset pool), scrubs cosmetic artifacts (bold-only headings,
   smallcaps/underline spans, empty headings, AI alt-text leaks,
   rights-boilerplate bounded to non-prose regions), and folds verse /
   lineated / dedication runs so stanzas render as verse blocks.
3. Hashes every body image (SHA-256, 12-char prefix) and dedups it, assigning
   each a content-addressed `./images/<hash>.<ext>` ref (relative to the work
   folder). Each deduped image is returned as a `PlannedAsset`
   (`lib/writeplan.py`); the converter copies nothing.
4. Lowers the IR to canonical Markdown exactly once (`lib/ir_lower.py`): no
   string round-trip, no regex tail-stripping. The footnote appendix is
   generated last from typed footnote definitions, so a definition can never be
   lost to tail-stripping.
5. Extracts cross-references from footnotes and inline mentions into frontmatter
   `cross_refs` (resolved-only, `{target: {kind, number}, …}`; `lib/cross_refs.py`).

The author-facing frontmatter (slug, number, title, tags, cover, translation,
description) is assembled by `import_docx.py` from existing Markdown frontmatter,
not by the converter — see the `import_docx.py` section above.

### Known gotchas

- Pandoc escapes `|` and `>` as `\|` / `\>` inside paragraphs (so a stray
  pipe doesn't get parsed as a GFM table cell). Markdown renderers render
  these as the literal characters; nothing to do.
- The first `![image1.png]` style image in some docx is sometimes the cover
  repeated inside the body. The canonical cover remains `cover.<lang>.<ext>`;
  if the repeated inline image is kept, it is treated like any other body
  asset and referenced as `./images/<hash>.<ext>` inside the work bundle.
- Imported body illustrations use stable work-local paths:
  `![](./images/<hash>.<ext>)`.
- A few books embed Russian-language alt text on AI-generated images — kept
  as-is on the rewritten Markdown image alt text.

### Requirements

- `pandoc` >= 3.x on `PATH` (`brew install pandoc`)
- `uv` with the repo `pyproject.toml` / `uv.lock` environment.

## docx_optimize.py

Shrinks the original `.docx` files for distribution as downloadable Word
documents. Source/destination mapping comes from
`data/conversion-manifest.json` provenance, not content frontmatter. The corpus
carries ~95% of its bytes as uncompressed 1024×1536 RGB PNGs from an AI image
generator; this script re-encodes those as JPEG q85 (visually
indistinguishable on phones/tablets) and rezips the result.

Full corpus: **1.1 GB → 267 MB (~77% saved)** in ~35 s on an M-series Mac.
The 65 MB outlier drops to 7.8 MB.

### Strategy

For each image in `word/media/`:

| Input                                                | Action                    |
|------------------------------------------------------|---------------------------|
| Tiny icon (< 30 KB and ≤ 256 px)                     | leave alone               |
| PNG, no alpha, large (≥ 512 px or ≥ 200 KB), smooth  | **JPEG q85 progressive**, rename `.png` → `.jpg` |
| PNG with alpha, palette, or sharp lines (high edge density) | re-save as optimized PNG |
| Existing JPEG, < 200 KB                              | leave alone               |
| Existing JPEG, ≥ 200 KB                              | re-encode q85 if it wins ≥ 5% |
| Long edge > 1600 px                                  | LANCZOS-downscale to 1600 first |

When an extension changes, the script rewrites the `Target=` attribute in
every `word/**/*.rels` and ensures `[Content_Types].xml` declares a
`<Default Extension>` for the new extension. Other parts of the docx
(`document.xml`, styles, headers, etc.) are copied verbatim.

### Output layout

Optimized source files land alongside their markdown siblings in the content
tree — not under `dist/`. Multi-source books keep part-N suffixes as source
artifacts; everything else is single-file `<lang>.docx`:

```
src/content/books/<slug>/{ru,en}.docx                    # single-source books
src/content/books/<slug>/{ru,en}-part{1,2,3}.docx        # multi-source books (02, 46)
src/content/poetry/<slug>/ru.docx
src/content/projects/<slug>/ru.docx
```

For a merged multi-source work, the public `<lang>.docx` download is produced
separately by `render_downloads.py --docx` from the merged Markdown. Do not
expose a part file as the per-work DOCX download.

The destination is computed from the central `data/conversion-manifest.json`
source provenance (with a markdown-frontmatter `original_filename` fallback for
poems and projects). Originals under `legacy/` are read-only.

### Usage

```sh
# Full corpus (skips outputs newer than their source).
uv run scripts/docx_optimize.py

# Force rebuild + per-image stats.
uv run scripts/docx_optimize.py --force --verbose

# Specific file(s) or directory; destination is still derived from src/content/.
uv run scripts/docx_optimize.py "legacy/books/ru/53-голос-еноха-...docx"

# Dry run.
uv run scripts/docx_optimize.py --dry-run
```

### Verification

```sh
unzip -t src/content/books/<slug>/ru.docx
textutil -info src/content/books/<slug>/ru.docx
qlmanage -t -s 1800 -o /tmp src/content/books/<slug>/ru.docx
```

### Tunables

Constants at the top of `docx_optimize.py`:

| Name | Default | Effect |
|------|---------|--------|
| `MAX_LONG_EDGE` | 1600 | Downscale anything bigger |
| `JPEG_QUALITY` | 85 | Subjectively transparent at ≤ 1600 px |
| `LARGE_PNG_MIN_EDGE` | 512 | Below this → keep PNG |
| `LARGE_PNG_MIN_BYTES` | 200_000 | Below this → keep PNG |
| `EDGE_DENSITY_DIAGRAM_MIN` | 0.13 | Above this → keep PNG |

### Known limitations

- Validated against macOS Quick Look + `textutil` (Apple's Office Open XML
  parser used by Pages/TextEdit/Preview). LibreOffice was not installed in
  this environment.
- No `mozjpeg`; we use Pillow's default JPEG encoder.
- Re-encoding PNG → JPEG drops any alpha. The heuristic only fires on PNGs
  without alpha, so this is safe; but if a future Word-emitted PNG carries
  alpha in a "photographic" image it will be kept as PNG.
- `dist/` is regenerable output — do not commit.

## render_downloads.py

Local admin tool that renders release artefacts into the work bundle, per the
contract in `docs/downloads.md`. PDF/EPUB outputs land as
`src/content/<kind>/<work>/<lang>.{pdf,epub}`. Merged multi-part DOCX outputs land
as `src/content/books/<work>/<lang>.docx` when `--docx` is passed. These files are
**committed**; CI never runs this script and never invokes pandoc or typst.

### Requirements

- `pandoc` (>= 3.0) on `PATH`
- `typst` (>= 0.13) on `PATH`
- `uv`

The Cyrillic-capable fonts and the typst template live in
`scripts/downloads-templates/` and `scripts/downloads-fonts/`. Both are
committed; no network access required.

### Usage

```sh
# Everything (cold run ≈ 1m40 on M-series for 148 PDFs + 101 EPUBs).
uv run scripts/render_downloads.py

# One work.
uv run scripts/render_downloads.py --book 33
uv run scripts/render_downloads.py --poem 1

# One language only.
uv run scripts/render_downloads.py --lang en

# Selective formats.
uv run scripts/render_downloads.py --skip-pdf
uv run scripts/render_downloads.py --skip-epub

# Merged DOCX release artifact for a multi-part work.
uv run scripts/render_downloads.py --book 2 --docx --skip-pdf --skip-epub --force

# Force regeneration even if the output is newer than the source.
uv run scripts/render_downloads.py --force
```

### Output policy

EPUB is books-only per `docs/downloads.md` scope. Poems and projects skip EPUB.

The script is incremental by default — it skips outputs newer than their
source `<lang>.md`. Use `--force` to rebuild everything.
